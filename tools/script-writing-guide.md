# 스크립트 작성 기준서

> 대화 스크립트를 교육 컨텐츠로 만들기 위한 작성 기준.
> script-format-guide.md(포맷)과 함께 사용.

---

## 원칙: 모든 세트는 교육 단위다

세트 1개 = **학습 목표 1개**. 대화가 아무리 자연스러워도 학습자가 뭘 배우는지 불명확하면 실패.

---

## 1. 세트 설계 순서

```
① 학습 목표 정의 (이 세트에서 뭘 배우나?)
② 상황 설계 (어떤 상황에서 이 표현이 필요한가?)
③ AI 대사 작성 (학습 목표를 자연스럽게 유도)
④ User 대사 작성 (학습 목표 표현 + `/` 선택지)
⑤ 검증 (AI 다음 턴이 양쪽 `/`에 다 자연스러운가?)
```

---

## 2. 학습 목표 4가지 유형

도민/민지에서 추출한 학습 목표 분류:

| 유형 | 설명 | 도민 예시 |
|------|------|----------|
| **표현 (Expression)** | 특정 상황에서 쓰는 핵심 표현 | 밥 먹었어요?, 수고하셨습니다, 잘 먹었습니다 |
| **문법 (Pattern)** | 반복 가능한 문법 패턴 | -어 봤어(경험), -ㄹ 수 있어(가능), -ㄹ게(약속), -것 같습니다(의견) |
| **기능 (Function)** | 대화에서 하는 "행위" | 사과하기, 제안하기, 거절하기, 칭찬 받기, 위로하기 |
| **어휘 (Vocabulary)** | 주제별 핵심 단어 | 카드/현금(결제), 매워요/덜 맵게(맵기), 떨려요/신나요(감정) |

**좋은 세트 = 최소 1개 유형이 명확**. 최고의 세트는 2개 이상 겹침.

도민 TR-2 세트3 예시:
```
학습 목표: [문법] 정도 표현 (많이/얼마나) + [기능] 요청 수정하기
AI: 비빔밥 좀 매워요. 괜찮으세요?
User: 많이 매워요? / 얼마나 매워요?     ← 정도를 묻는 두 가지 방법
AI: 조금 매워요.
User: 덜 맵게 해주세요. / 그럼 괜찮아요!  ← 수정 요청 vs 수락
```

## 3. User `/` 선택지 품질 기준

| 등급 | 기준 | 예시 | 판정 |
|------|------|------|------|
| **A** | 다른 대화 기능 | 수락 vs 거절, 질문 vs 반응, 사과 vs 변명 | ✅ 최우선 |
| **B** | 같은 기능, 다른 문법/어휘 | 거의 다 왔어 vs 나가고 있어 (진행 상태), 많이 매워요? vs 얼마나 매워요? | ✅ 좋음 |
| **C** | 같은 의미, 강도만 다름 | 좋아해 vs 진짜 좋아해, 떨려요 vs 진짜 떨려요 | ⚠️ 최소화 |
| **D** | 완전 동의어 | 대박 vs 미쳤어, 네 vs 응 | ❌ 지양 |

**목표: A+B가 전체 `/` 선택지의 70% 이상**

도민 벤치마크:
- A등급 (다른 기능): ~47%
- B등급 (다른 문법/어휘): ~18%
- C등급 (강도 차이): ~17%
- 선택지 없음: ~35% (상당수는 문화 표현 고정구로 불필요)

---

## 4. 선택지 없어도 되는 경우

| 상황 | 이유 | 예시 |
|------|------|------|
| **문화 고정 표현** | 변형하면 부자연스러움 | 수고하셨습니다, 잘 먹었습니다, 안녕히 가세요 |
| **이름/개인정보 입력** | ___로 처리 | ___이에요, ___살이야 |
| **결(마무리) 세트** | 작별 인사는 패턴이 고정 | 잘 자, 내일 봐, 또 봐 |
| **감정 클라이맥스** | 선택지가 몰입을 깨뜨림 | 보고 싶어, 미안해. 밥 사줄게 |

## 5. AI 대사 작성 기준

```
AI 대사 = [리액션] + [유도]
```

| 요소 | 역할 | 좋은 예 | 나쁜 예 |
|------|------|--------|--------|
| **리액션** | User 대사에 자연스럽게 반응 | 오, 진짜? / 대박! / 그렇구나~ | (없음) |
| **유도** | 다음 User 대사를 자연스럽게 끌어냄 | 너는? / 뭐 먹고 싶어? / 어떻게 생각해요? | 네! / 좋아요! (대화 끊김) |

**AI가 질문이나 제안으로 끝나야 User가 말할 이유가 생긴다.**

나쁜 AI:
```
AI: 네! 다행이에요!     ← 대화 끊김. User가 뭘 말해야 하는지 모름
```

좋은 AI:
```
AI: 다행이에요! 다음엔 제가 더 잘 할게요!   ← User가 반응할 거리가 있음
```

## 6. 세트 간 흐름 기준

```
세트 N의 마지막 대사 → 세트 N+1의 첫 대사가 자연스럽게 이어져야 함
```

Outro가 브릿지 역할:
```
세트 3 Outro: "Now, prepare for the ultimate Korean obsession... Food."
→ 세트 4 AI: "한국 음식 먹어 봤어?"
```

**에피소드 내 흐름 체크:**
- 기(세트1-2): 상황 설정, 인사, 기본 정보 교환
- 승(세트3-4): 주제 심화, 새로운 표현 등장
- 전(세트5): 전환점 or 클라이맥스
- 결(세트6): 마무리, 다음 약속, 작별

## 7. 테마별 학습 목표 배분

| 테마 | 학습 목표 중심 | 주요 유형 |
|------|-------------|----------|
| **PI** | 자기소개, 취미, 약속, 일상 대화 | 표현 + 기능 |
| **CR** | 직장 에티켓, 보고, 요청, 사과 | 표현 + 문화 |
| **KD** | 감정 표현, 리액션, 위로, 갈등 해결 | 기능 + 어휘 |
| **KP** | 팬 인사, 칭찬, 격려, 감사 | 표현 + 문화 |
| **TR** | 주문, 결제, 길찾기, 요청 | 패턴 + 어휘 |

## 8. 검증 체크리스트

세트 작성 후 반드시 확인:

```
□ 이 세트의 학습 목표를 한 문장으로 말할 수 있는가?
□ User `/` 선택지가 A 또는 B 등급인가?
□ AI 다음 턴이 User의 양쪽 선택지에 다 자연스러운가?
□ AI 대사가 질문 또는 제안으로 끝나는가? (결 세트 제외)
□ 이전 세트에서 이 세트로 자연스럽게 이어지는가?
□ A0 학습자가 이 표현을 실제 상황에서 쓸 수 있는가?
□ Outro가 다음 세트로의 브릿지 역할을 하는가?
□ 같은 표현이 에피소드 내에서 3회 이상 반복되지 않는가?
```
